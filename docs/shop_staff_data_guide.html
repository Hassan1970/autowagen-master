Public shop — where the data comes from

Customers see autowagen-master/shop/ (no login). Everything they see is read from your MySQL database — mainly the parts table and links you maintain in the staff app.

2. Checklist — put one part on the web

  1. Log in to the staff site → Inventory → Add part (or edit an existing part).
  2. Set Part name (this is what customers read — no need to put internal stock code in the title).
  3. Set Qty on hand, Status = Available, Asking price.
  4. In the EPC section, link the part to a variant under Engine (or another category) if you want it under that shop area.
  5. Tick List on public website (Stage 6b — needs sql/06b_web_shop.sql applied once on the database).
  6. Upload photos on the part if you can.
  7. Save. Open the public shop in a private browser window and search or use Browse parts / Engines.

3. Engines menu — three “lanes” (optional)

The shop Engines dropdown splits listings using words in the part name. Staff should use clear, consistent titles.

Engine parts (loose) — ordinary spares (radiators, pumps, etc.). Avoid assembly phrases in the name.

Complete engines / sub-units — include phrases like: complete engine, long block, long motor, full engine, engine assembly, sub unit.

Cylinder heads / bare — include phrases like: cylinder head, cyl head, bare head, head bare.

Rules are defined in code: includes/shop_helpers.php → functions shop_normalize_engine_line / shop_engine_line_sql_condition. For a stricter system later, a dedicated field on parts could be added — speak to your developer.

4. Internal stock codes (SKU)

Staff still use SKU in Inventory (e.g. AWG-0002-P01, OEM-0004). The public shop hides SKU from customers on listing and detail pages; it remains in the database for invoices, orders, and staff search.

5. Databases: local vs live

  • Laragon (PC): database name is usually autowagen_master — for testing.
  • Hosting: same PHP code, but your host’s database name and config/secrets.live.php. Data entered on the PC does not sync automatically to the website — export/import or re-enter on live when you go live.

6. SQL you may need (once per database)

  • sql/06b_web_shop.sql — public shop columns + order tables.
  • sql/06e_shop_guest_enquiries.sql — guest Message inbox.
  • Stage 2 EPC + epc_full_view — needed for engine category browsing.
