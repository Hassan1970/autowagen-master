§1 — Accounts payable (who we still owe)

What it is for

You bought parts or wreck stock via Supplier purchases and entered a Bill (ZAR). This report lists every purchase row where bill − payments > 0. It answers: “Who must we pay, how much is left?” — includes registered suppliers and private sellers on purchases.

How to open it (logged in)

  1. Ensure Laragon shows Start All (Apache + MySQL green).
  2. Browser → login: http://localhost/autowagen-master/auth/login.php
  3. Black top menu → click Reports → click Accounts payable (owed).

Shortcut URL: …/supplier_ap_report.php

What you can do on screen

  • As at (report date) — pick a date and Apply (for your records; balances are “current” open amounts).
  • Print / PDF — red button prints a clean sheet (navigation hidden). Requires current supplier_ap_report.php deployed to your server.
  • What A–D means — toolbar link opens docs/supplier_ap_report_explained_print.html (browser → Ctrl+P → Save as PDF).
  • Total still owed — red summary card at top.
  • Table columns: Purchase #, Owed to (supplier name or private seller), Bill, Paid so far, Balance, Status (Unpaid / Part paid), Due, Open → goes to that purchase to add payments.
  • Summary by who we owe — small cards grouped by party.
  • Link Supplier purchases → list of all purchase batches.

If the page warns about SQL

Run sql/04d_supplier_accounts_payable.sql in phpMyAdmin on database autowagen_master (after 4c). Until then the report may show a yellow warning.

§2 — Accounts receivable (customers who owe us)

What it is for

After POS you Finalize invoices. If the customer did not pay in full, a balance remains. This report answers: “Which customers owe us money, which invoices are overdue by due date?”

This is NOT supplier PAYABLE. If you are looking at purchase batches (HASSAN NIZAMIE, Purchase #2), use §1 Accounts payable instead.

How to open it

  1. Black menu → Reports → Accounts receivable (owed).

Shortcut URL: …/customer_ar_report.php

Printable zones A–F + example layout: docs/customer_ar_report_explained_abcdef_print.html (double-click → Ctrl+P → Save as PDF).

What you can do on screen

  • As at — date used to mark overdue (invoice due date strictly before this date, with unpaid balance).
  • Ticks: Overdue only narrows lists. If your database ran 06a: Account customers only restricts to flagged account buyers.
  • Apply refreshes filters.
  • Print / PDF button — uses browser print (hide nav).
  • By customer — one row per customer: balance, overdue portion, links Statement and Customer.
  • By invoice — open balances per invoice with badges (Overdue Nd, Open, No due date).

If the page warns about SQL

  • Run sql/05_pos.sql for invoices/payments tables.
  • Account customers only tick needs sql/06a_customer_account.sql — otherwise AR still works without that filter.

§3 — Customer account statement

What it is for

Gives one named customer a printable breakdown: outstanding final invoices (totals, paid, balance), optional recent payments, and shortcuts to WhatsApp / PC email (summary text — you attach the PDF yourself after Print / PDF).

How to open it

Path A — Reports menu

  1. Menu Reports → Customer statements (Customers list).
  2. Find the person in the table → click Statement on that row.

Path B — Master data

  1. Menu Master data → Customers.
  2. Find the person in the table → click the Statement link on that row.

Path C — from AR

  1. Open §2 Accounts receivable.
  2. In By customer, click Statement beside that customer.

URL pattern: …/customer_statement.php?id=CUSTOMER_ID (browser shows number after opening once).

What you can do on screen

  • Statement as at + Include paid-in-full invoices tick → Apply.
  • Print / PDF → formal PDF for the client attachment.
  • WhatsApp / Email — prefilled summary (not full branded PDF).
  • Edit customer / link back to AR report.

§4 — Web shop orders

What it is for

Lists guest orders from the public /shop/ checkout. Stock was reduced when the order was placed. Use it to call the buyer, coordinate payment/collection, and cancel an order if needed (restores stock).

This is not a full “sales analytics” report (no monthly charts) — it is an operational order list.

How to open it

  1. Menu Reports → Web shop orders.

Shortcut URL: …/shop_orders_admin.php

What you can do on screen

  • List of WEB-YYYY-NNNNN orders with status (e.g. confirmed / cancelled).
  • Open a row → buyer name, phone, lines (SKU, qty, snapshot prices), totals.
  • Cancel order & restore stock — only when business rules allow; confirms in browser.
  • Link to open the public shop in a new tab.

If the page warns about SQL

Run sql/06b_web_shop.sql in phpMyAdmin.
