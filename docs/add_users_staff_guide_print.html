1 · What this guide covers

Autowagen Master does not have a “User management” screen in the web app yet. New logins are added by inserting (or editing) a row in the MySQL table users, usually via phpMyAdmin on hosting or on Laragon. Passwords are stored as a one-way bcrypt hash in password_hash — never paste plain-text passwords into that column.

After login, the top-right menu shows a small badge with the role (e.g. viewer). Navigation hides buttons this role is not meant to use.

3 · Which database?

  1. Local PC (Laragon): phpMyAdmin → database you use for dev (often autowagen_master). Secrets live in config/secrets.local.php.
  2. Live website: phpMyAdmin from your hosting control panel → database named in config/secrets.live.php → key db.name. Must match where you insert the user.

Important: The table user_login_attempts only logs attempts — it does not hold passwords or roles. Always use table users.

4 · Generate a password hash (Laragon terminal)

You need one line starting with $2y$10$ … for column password_hash.

  1. Open Laragon → right-click tray icon → Terminal (or use Cmder).
  2. Type this as one line. Replace YourPasswordHere with the real password the person will type at login (keep the single quotes around it):
    php -r "echo password_hash('YourPasswordHere', PASSWORD_DEFAULT), PHP_EOL;"
  3. Press Enter. Copy the entire output line (no spaces cut off at the start).

Common mistake: The second argument must be the word PASSWORD_DEFAULT, not the password again. Wrong: password_hash('x', x).

If Windows says php is not recognized, run PHP by full path, for example adjust version folder under C:\laragon\bin\php\.

5 · Add a new user — phpMyAdmin (numbered clicks)

  1. Log into phpMyAdmin (local or hosting).
  2. In the left sidebar, click your Autowagen database (same name as secrets.*.php → db.name).
  3. Click the table users.
  4. Open the tab Insert.
  5. Fill in:
    ColumnWhat to enter
    usernameLogin name — unique, no spaces recommended (e.g. sarah, viewer).
    full_nameDisplay name (e.g. Sarah Jones).
    emailOptional; may be empty / NULL if your table allows.
    password_hashPaste the full $2y$10$… line from section 4.
    roleChoose one: owner, admin, manager, staff, or viewer.
    is_activeSet to 1 so login works.
  6. Leave id empty if phpMyAdmin says it auto-increments.
  7. For created_at / updated_at, use defaults such as CURRENT_TIMESTAMP if offered.
  8. Click Go. You should see a green success message.

6 · Change an existing user’s password or role

  1. phpMyAdmin → database → users → Browse.
  2. Click Edit (pencil) on that row.
  3. To reset password: generate a new hash (section 4), paste into password_hash, click Go.
  4. To make someone read-only: set role to viewer, save.
  5. To disable login without deleting history: set is_active to 0.

7 · “Too many failed attempts” (locked out)

The login screen counts wrong passwords per username (stored lowercase in the log) and IP address. After six failed tries within a rolling fifteen-minute window, the next attempt is blocked until that window passes. A successful login clears earlier failures for that username+IP. If you are testing or need an immediate reset, clear the log table (below).

  1. phpMyAdmin → correct database → tab SQL.
  2. Paste:
    DELETE FROM user_login_attempts;
  3. Click Go. Retry login with the correct username and password.

8 · Test the new login

  1. Open your site login page (e.g. …/auth/login.php).
  2. Use username exactly as in users.username.
  3. Use the plain password you typed inside password_hash('…', PASSWORD_DEFAULT) — not the long hash.
  4. Confirm the role badge top-right matches what you set.

10 · Security checklist (short)

  • Never commit secrets.live.php or real passwords to Git.
  • Use strong passwords for staff accounts; share verbally or password manager — not email.
  • viewer is for read-mostly access; do not give owner unless required.
  • After testing on live, rotate passwords if they were shown on screen or written in chats.
