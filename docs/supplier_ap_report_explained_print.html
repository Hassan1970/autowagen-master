Autowagen Master — Accounts payable report (staff guide)

Matches Reports → Accounts payable (owed) (supplier_ap_report.php). This file is plain HTML — save as PDF: Ctrl+P → Save as PDF · path: docs/supplier_ap_report_explained_print.html

Lost the Print button on the live site? The app only shows Print / PDF after you upload the latest supplier_ap_report.php from your PC to your host (same folder as on Laragon). See the dashed box below.

Deploy so the Print button appears on production
  1. On your PC: file autowagen-master/supplier_ap_report.php (in Cursor / File Explorer).
  2. Upload it with FTP / cPanel File Manager to your website folder, overwriting the old file → path must end like …/supplier_ap_report.php next to other PHP pages.
  3. Refresh https://your-domain/supplier_ap_report.php with Ctrl+F5. You should see Print / PDF and this guide link on the toolbar.

A — Top total & “As at” date

A1 — Red banner (“Total still owed…”) adds up every unpaid balance from the rows below — that is money your business still owes suppliers or private sellers on recorded bills.

A2 — As at (report date) is the date printed on the report for your records. The money columns are current open balances from the database (not a historical “what we owed on that old day”).

B — Main table (each row = one supplier purchase)

One row appears for each Supplier purchase batch where Bill − payments > 0
  • Purchase — internal number #… plus your optional reference text (invoice no., etc.).
  • Owed to — supplier name from Master data → Suppliers, or a private seller name from the purchase.
  • Bill (ZAR) — total bill amount you typed on Inventory → Supplier purchases → Edit purchase.
  • Paid so far — sum of all active payment lines on that purchase.
  • Balance — left to pay: Bill − Paid so far.

C — Status & Due columns

  • Unpaid (red badge) — no payments recorded yet (or effectively zero).
  • Part paid (yellow badge) — some payments exist but Balance is still positive.
  • Due — shows the purchase’s due date if you saved one on the purchase; — means no due date on file.
  • Open (button) — opens that purchase to add more payments or fix the bill. (Hidden on paper when you use Print / PDF.)

D — “Summary by who we owe”

This section groups balances by party (each supplier gets one card; each private seller with their own purchase may appear separately). Adding those card totals should match the rows above for each name.

Making a PDF from the report screen

  1. Open the payable report while logged in.
  2. Click the red Print / PDF button (after deploy — see dashed box).
  3. In the dialog: destination Save as PDF → Save.
  4. If red bars print grey: tick Background graphics in print settings (Edge / Chrome).

Accounts payable ≠ accounts receivable

  • Accounts payable (this report) — You owe them (supplier / seller).
  • Accounts receivable — Customers owe you after sales invoices. Different menu line.
