Phase A — Live / ops

  1. PC: save work · git push · (docs/git_laragon_terminal_start_to_finish_print.html).
  2. Upload or pull program files to hosting (FTP / cPanel / server git). If menus are old: at least includes/header.php, main_dashboard.php, plus any files you changed.
  3. Server: config/secrets.live.php = live DB (copy from secrets.live.php.example — never commit real passwords).
  4. Host phpMyAdmin: run missing sql/*.sql in order for that database (see CLAUDE.md section 10).
  5. Browser: open live URL → login → Reports → one report loads.

Phase B — Test (after live is updated)

  1. Login · Dashboard.
  2. POS: New sale or Sales invoices — open a draft.
  3. Reports: e.g. AR + Sales summary — no error page.
  4. Guest: /shop/ on the live base URL.
  5. Optional: Credit notes if 07 ran on live DB.
  6. Log PASS or issue in CLAUDE.md section 9.

Phase C — New code (backlog — one block at a time)

Only after A + B, unless you deliberately build on Laragon only first.

Phase D — Test (after each backlog feature)

  1. Test changed screens on Laragon.
  2. git commit · git push.
  3. Deploy to live (Phase A file/SQL steps as needed).
  4. Smoke-test live again (Phase B + feature-specific checks).
  5. Update CLAUDE.md section 9 and docs/CHANGELOG.md / CHANGELOG.html.
