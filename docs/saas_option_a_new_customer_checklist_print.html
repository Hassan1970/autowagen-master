OPTION A — New customer (one tenant = one database)

Valatone SaaS provisioning · repeatable manual checklist · timezone Africa/Johannesburg UTC+2 · pair with docs/VALATONE_SAAS_WAY_FORWARD.md
Save as PDF: Chrome or Edge → Ctrl+P → Save as PDF → Pages: All → Background graphics ON. Clear selection first so print is not “selection only”.

What Option A is: Each yard gets its own MySQL database, its own secrets.*.php, and its own deployed folder/subdomain. No shared multi-tenant table — isolation is by separate DB.

Never point two customers at the same database name. Never reuse another client’s FTP path unless you intentionally overwrote them (don’t).

Per customer — do in order

  1. Record (write on this sheet footer or your CRM): company name · planned URL/subdomain · DB name · DB user · provision date · who ran the checklist.
  2. Create an empty database on the server (hosting control panel or phpMyAdmin) with UTF-8 (utf8mb4). Example name pattern: yard_slug_master — unique per customer.
  3. Create MySQL user granted only that database — not global root on production.
  4. Deploy PHP files to this customer’s directory (FTP, git deploy, ZIP). Same codebase version as your gold master.
  5. Secrets file on the server: config/secrets.live.php — set db.name, db.user, db.pass, db.host, and app.url to their exact public base URL (trailing slash rule = match whatever the app expects). Never commit secrets to Git.
  6. Run SQL migrations on this database only — phpMyAdmin → pick that DB → SQL tab → paste file contents → Go. Use the table in the next section in order. Skip a file only if you are absolutely sure it already ran on this DB.
  7. First login user: insert one row into users with a bcrypt password_hash, role owner — full steps: docs/add_users_staff_guide_print.html.
  8. Smoke test: open their URL → login → Dashboard · create one test vehicle or part (optional) · open POS draft · open public /shop/ (if used) · Reports menu loads.
  9. Handoff: give the customer their login URL + username reset policy; keep your copy of DB name and deploy path.
