§3 · Hosting server — what we said about “connecting”

GitHub is not your live website by itself. It only stores code when you push. Customers see a host (shared hosting/VPS).

Three separate places (don’t mix them up)

PlaceWhat happens there
Your PC (C:\laragon\www\…)You edit files; Cursor save = instant on disk for localhost.
GitHubBackup + history — only updates when git push.
Live hostNeeds FTP/File Manager upload or git pull on server (if you have SSH). DB is separate phpMyAdmin on host.

Typical workflows

  • Only cPanel/File Manager/FTP: After you finish changes locally (and optionally push to GitHub for backup), upload changed PHP/files to hosting.
  • SSH + Git on server: git clone once per server folder, later git pull after each git push from home.
  • Cursor “Remote SSH”: only if your hosting package gives SSH and you deliberately open that remote folder — optional advanced path.

§5 · Git + GitHub — why bother?

  • Undo / history: every commit is a labelled snapshot.
  • Off‑site backup of code when you push.
  • New PC later: git clone … gets the folder back.
  • Secrets & uploads not in GitHub: project .gitignore skips config/secrets.local.php, config/secrets.live.php, and files under uploads/ (keeps only uploads/.htaccess for safety rules).

Saving files in Cursor does not upload to GitHub. You run git add → commit → push when you want a backup online.

You don’t have to push after every single line of code — do it after a sensible chunk (end of evening, bugfix done, etc.).

§9 · Problems we solved today — cheat sheet

  • Vim opened instead of committing: you ran bare git commit. Fix: Esc → type :q! → Enter → then git commit -m "…" so Vim stays closed.
  • Vim line “14,1”: just cursor position — meaningless for which files Git commits.
  • Green “new file” in git status: staging worked; missing step was only commit then push.
  • Cursor main* stuck: Ctrl+Shift+P → Reload Window after a push.
  • Microsoft Print dialog “doesn’t support print preview”: still try Print to PDF — or print from Chrome/Edge with the HTML file open.

§10 · Reboot · new AI chat · different project

  • PC restart: Git stays installed; .git stays in folder — don’t rerun git init inside this project unless you delete .git.
