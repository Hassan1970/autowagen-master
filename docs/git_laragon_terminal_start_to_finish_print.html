Part A — Open the terminal

  1. Laragon: Right-click the Laragon tray elephant → Terminal or open Cmder if you use it.
  2. Cursor: Menu Terminal → New Terminal or press Ctrl+` (backtick).
  3. Go to the project folder (type exactly, then Enter):
    cd C:\laragon\www\autowagen-master
  4. Check you are in Git:
    git status
    If you see branch name + file list → OK. If not a git repository → wrong folder; fix the cd path.

Part B — Save work on GitHub (every session you care about)

Three commands in order: stage → commit → push.

  1. Stage everything (all saved files in the folder):
    git add .
  2. Commit with message on the same line (avoids Vim opening):
    git commit -m "Short plain English: what you changed"
    If it says nothing to commit, working tree clean → there were no new changes since last commit (still run git push if unsure).
  3. Push to GitHub:
    git push

Part C — First time Git asks for login (HTTPS)

PromptWhat to type
UsernameYour GitHub username (same as in github.com/yourname/…).
PasswordNot your GitHub website password. Use a Personal Access Token (next section).

Part D — Create a Personal Access Token (once per token)

  1. Browser → GitHub → profile photo → Settings → Developer settings → Personal access tokens → Tokens (classic) → Generate new token (classic).
  2. Note: any unique label (e.g. Laragon May 2026). If GitHub says Note has already been taken, change the text.
  3. Tick scope repo (full control of private repositories).
  4. Generate token → copy the token (ghp_…). Store it safely — GitHub shows it once.
  5. Never paste the token into chat or screenshots.

Part E — If push fails

MessageWhat it means / fix
Password authentication is not supported / Invalid username or tokenUse token as password, not site password. Or generate new token.
Everything up-to-dateRemote already has your commits. If you expected new files → run git status; if files listed → do Part B (add, commit, then push).
Git keeps wrong passwordWindows Credential Manager → Windows Credentials → remove git:https://github.com → git push again → username + token.

Part F — If Vim opens after git commit

You ran git commit without -m "…". Options:

  • Abort commit: Esc then type :q! Enter.
  • Finish commit: i, type message line, Esc, :wq Enter.

Habit: always use git commit -m "your message" so Vim does not open.

What Git does not backup

  • MySQL database — export with phpMyAdmin separately.
  • Files listed in .gitignore (e.g. local secrets, most of uploads/) — by design.
