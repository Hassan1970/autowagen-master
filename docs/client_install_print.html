AUTOWAGEN MASTER

Install on a client computer (Windows + Laragon) — staff sheet
How to save this as a PDF: Press Ctrl+P (or menu Print). Choose Save as PDF or Microsoft Print to PDF. In Edge/Chrome you may need More settings → turn on Background graphics if colours do not print.

This app runs in a browser on the client PC. You need PHP 8, MySQL, and the project files. The easiest bundle on Windows is Laragon.

1. Install Laragon on the client PC

  1. Download Laragon from laragon.org and install it.
  2. Start Laragon and click Start All so Apache and MySQL run.
  3. Use PHP 8.x (Laragon menu → PHP).
  4. For Site backup (ZIP) in the program: open PHP → php.ini, find extension=zip (remove the leading ; if present), save, then restart Apache.

2. Copy the project folder

  1. Copy the whole folder autowagen-master to the client machine, for example:
    C:\laragon\www\autowagen-master\
  2. Copy everything (PHP files, sql, config, includes, auth, uploads/.htaccess, etc.).

3. Create the database

  1. Open http://localhost/phpmyadmin
  2. Create database autowagen_master (collation utf8mb4 if asked).
  3. Open the SQL tab. Run each script from the project’s sql\ folder once, in this order (skip any already applied):
OrderFileNote
101_auth.sqlUsers / login
202_epc.sqlEPC tree; optional extra 02b…02h if you use them
303_master_data.sqlVehicles, customers, suppliers
403b_vehicle_extras.sqlVehicle fields
503c_proof_of_residence.sqlVehicle proof of residence
603d_customer_compliance.sqlCustomer SHGA docs
704_inventory.sqlParts
804b_part_tpp_compliance.sqlTPP columns
904c_supplier_purchases.sqlPurchases
1004d_supplier_accounts_payable.sqlAP
1105_pos.sqlInvoices / POS
1206a_customer_account.sqlAccount customers (optional but recommended)

If you are importing a full SQL dump from another PC instead, use phpMyAdmin Import on autowagen_master and skip re-running scripts that are already in the dump.

4. Configure config/secrets.local.php

  1. On the client PC there must be a file: config/secrets.local.php (this file is not in Git — create it on each machine).
  2. Copy from config/secrets.live.php.example or another machine’s template. It must return PHP array keys db and app.
  3. Set MySQL: host (usually localhost), name = autowagen_master, user, pass (Laragon default is often user root, empty password).
  4. Set app.url to the URL used in the browser, e.g. http://localhost/autowagen-master (no trailing slash — see your main CLAUDE.md).
  5. Set app.debug to true only while testing on local.

Never email secrets.local.php in plain text or commit it to a public repo.

5. Open the app and log in

  1. Browser: http://localhost/autowagen-master/ (or the URL you set in app.url).
  2. Use the owner account created by 01_auth.sql (your project may use username hassan — use whatever was set when the database was built).

6. Uploads and permissions

  • Folder uploads/ must exist and be writable so photos and scans save correctly.
  • Keep uploads/.htaccess in place for security.

7. Moving data from your PC to the client’s

  1. Files: In the program (owner/admin): Dashboard → Site backup → create ZIP, download. Unzip on the client under www\autowagen-master or merge uploads as needed.
  2. Database: On your PC: phpMyAdmin → database autowagen_master → Export → SQL. On client: Import that file into their autowagen_master.
  3. On the client, still create or edit secrets.local.php for their MySQL and URL.
