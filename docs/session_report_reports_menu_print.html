1 — Summary (what changed)

A new Reports dropdown was added to the black staff top bar in includes/header.php. It groups list/report-style screens in one place. Items that were duplicated under Inventory or POS were removed from those menus so there is a single obvious path for report-style pages.

2 — Database / SQL on live server

No new tables and no ALTER statements were shipped for this change. Live behaviour is updated by deploying the updated PHP (and optional HTML docs) only. Your MySQL schema is unchanged by this Reports-menu work.

If you use guest web shop messages (shop_enquiries_admin.php), that feature still needs sql/06e_shop_guest_enquiries.sql run once per database — that is separate from the Reports menu and was not part of this nav change.

5 — How to log in as viewer

Roles live in the users table (role enum includes viewer). There is no separate “viewer login page” — same auth/login.php as everyone else; the role comes from the row for that username.

Option A — New user (phpMyAdmin)

  1. Open phpMyAdmin → database autowagen_master (or your live DB name) → table users → Insert.
  2. Set username (unique), full_name, leave email null or unique, set role = viewer, is_active = 1.
  3. For password_hash you must store a PHP password_hash value (not plain text). On a PC with PHP: run
    php -r "echo password_hash('YourPasswordHere', PASSWORD_DEFAULT);" Copy the full hash string into the password_hash column, then save the row.
  4. Log out of Autowagen if needed → open login URL → sign in with that username/password. Your name badge should show viewer.

Option B — Temporarily downgrade an existing user (testing only)

  1. phpMyAdmin → users → Edit the row → set role to viewer → Go.
  2. Log in as that user. (Remember to set the role back when finished testing.)

6 — Features not built (clarification)

Customer returns / credit notes, supplier returns, automated sales analytics PDF from the app, and similar items remain out of MVP unless you schedule a dedicated build. The Reports menu only reorganizes existing pages; it does not add returns or a new sales report module.
