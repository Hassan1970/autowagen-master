Engine data & public shop — reference notes

Autowagen Master — where engine-related data lives, how subcategories work, and how the website reads it. For owners and staff keeping documentation.

1. Where the program reads data from

All shop pages use PHP + PDO (config/config.php) to read one MySQL/MariaDB database (on Laragon usually autowagen_master; on hosting, the name in secrets.live.php). There is no separate “engine server” — engines on the site are normal inventory rows plus EPC links.

2. EPC tree — what “Engine” and subcategories really are

The catalogue is a fixed-depth tree (Stage 2). Each part can be linked to one or more variants (bottom level). The shop’s Engines section keeps any part whose link path includes the top category with slug engine.

Six levels (top → bottom)

  1. Category — e.g. Engine (epc_categories.slug = engine)
  2. Subcategory — e.g. Petrol, Diesel (epc_subcategories)
  3. Type — e.g. Inline 4, V6 (epc_types)
  4. Subsystem — e.g. Cooling, Lubrication (epc_subsystems)
  5. Component — e.g. Radiator (epc_components)
  6. Variant — e.g. OEM, Aftermarket (epc_variants) ← what you pick for the part

Example paths from the seed SQL (sql/02_epc.sql)

Engine └── Petrol └── Inline 4 ├── Cooling │ └── Radiator → OEM | Aftermarket └── Lubrication → … Engine └── Diesel → … Brakes / Body … (other top categories, not “engine” slug)

Your live tree may be larger if you ran optional expansion scripts (02b–02h). More subcategories = more precise tags, but the shop Engines page still groups everything whose category_slug is engine, regardless of Petrol/Diesel/Cooling, unless you add more shop filters later.

3. How a part gets into “Engines” on the website

  1. Part exists in parts with is_active, status = available, qty_on_hand > 0, list_online = 1.
  2. Row exists in part_epc_links linking part_id to some variant_id.
  3. That variant joins through epc_full_view to a row where category_slug = 'engine'.

PHP file: shop/category.php — SQL joins parts → part_epc_links → epc_full_view and filters efv.category_slug = :cat_slug (for Engines, engine).

4. Engines menu — three extra “lanes” (same Engine category)

Besides All engine listings, the dropdown filters by words in parts.name (not by EPC subcategory level). Staff must use consistent titles.

To drive lanes from EPC instead of names in the future, you would add a field on parts (e.g. assembly class) or map EPC components — that would need a small schema + code change.

7. Staff checklist (engines on the web)

  1. Edit part → set name, price, qty, available.
  2. Link EPC path that ends under Engine (any valid sub-path you use).
  3. Tick List on public website.
  4. For submenu lanes: set part name wording per section 4.
  5. Upload photos; save.
