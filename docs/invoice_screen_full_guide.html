Sales invoice screen — full guide

Illustrated version: save PNGs into docs/manual_screenshots/ per docs/TRAINING_SCREENSHOTS.md, then uncomment the <img> lines in this file.

Save as PDF: Chrome or Edge → Ctrl+P → Save as PDF.
Open this screen: POS → New sale (creates a draft) or Sales invoices → Open on a row. Add parts: button Select item… (search + pick row) or numeric Part ID box or from All parts (see zone P) when opened from the modal links.

Invoice states (draft → final → void)

  • Draft — You can change customer, dates, notes, add/remove lines. Nothing has been taken from stock yet.
  • Final — Invoice number assigned (e.g. INV-2026-00001). Part-linked stock is reduced; parts may become Sold. You can record Payments.
  • Void — Cancelled for accounting; stock is not automatically put back — fix parts manually if it was a mistake.

Screen map (draft — match your screenshots)

Each zone below has a grey Figure box: that is where your PNG goes (filename shown). Until you add files, print/PDF still works as a text-only manual.

TOP

Title row & link

What: Main heading (Draft invoice #1 or Invoice INV-…), one-line hint under it, and All invoices back to the list.

Why: You always know which record you are in and how to return to the list.

When: Every visit.

How: No save — navigation only. Green/red alerts may appear below after an action (save, finalize, error).

A

Customer & dates (+ Save draft)

What: Customer dropdown (active customers from Master data). Invoice date (tax/document date). Due date (on account) — optional; use when the buyer pays later so you know when money was expected. Notes. Red Save draft.

Why: The customer is required before Finalize. Due date is stored for your process (future: overdue / reminders — not auto-email yet). Notes stay on the invoice row in the database.

When: While the invoice is a draft. After finalizing, this block becomes a read-only summary (same info, no edit).

How: Choose customer and dates → click Save draft. You must save a customer before finalize will work.

Figure. Customer dropdown, invoice date, due date, notes, Save draft.

Save as: docs/manual_screenshots/invoice-01-customer-dates.png

B

Lines

What: Table of everything you are selling on this invoice: description, qty, amounts ex VAT, VAT, column total incl VAT per line. In the card header, a large bold badge shows the running total incl. VAT for the whole invoice (updates every time you add or remove a line). In draft, each row can have × to remove.

Why: This is the legal/commercial breakdown of the sale. Totals feed Finalize and later Payments (what the customer owes).

When: Build lines in draft; after final they are fixed (no delete from here).

How: Add rows with S (search modal), P (full parts list from invoice), C (part ID), or D (manual line). SHGA badge on a row means second-hand / stripped buyer rules — see F.

Figure. Lines table + header badge showing R … incl. VAT (large / bold on screen).

Save as: docs/manual_screenshots/invoice-02-lines-running-total.png

S

Select item… (search modal)

What: Opens from the red Select item… button (draft, permission to edit). Type part name, SKU, or vehicle — results show only Available stock. Click a row to load Part ID, Qty, and Price ex VAT into the add form on the page; then click Add to invoice (or equivalent) on the main page.

Why: Faster than memorizing numeric IDs. You can still override price before adding (e.g. discount).

When: Draft only.

How: Select item… → type → pick row → set qty/price if needed → confirm add on the invoice. Optional: coloured links (Stripped, etc.) open All parts filtered — see P.

Figure. Modal open with search box + at least one result row.

Save as: docs/manual_screenshots/invoice-03-select-item-modal.png

P

All parts from invoice (Back + Add to invoice)

What: From the modal, links like Stripped open Inventory → All parts in the same tab with return=invoice_edit.php?id=…. A blue bar shows Back to invoice. On a draft, the table may include an Invoice column: Qty + Add to invoice posts straight to the draft (same rules: part must be Available, qty ≤ on hand). Search and pagination keep return.

Why: Lets staff browse the full list (including not-yet-available rows) and jump back without relying only on browser Back.

When: Draft only for Add to invoice; Back works anytime you used the modal link.

How: Click coloured source link → find part → if Available, set qty → Add to invoice. If status is e.g. On vehicle, edit the part first to Available.

Figure. Parts list with blue Back to invoice and at least one Add to invoice control.

Save as: docs/manual_screenshots/invoice-04-parts-admin-return.png

C

Add part by numeric Part ID

What: Fields Part ID, Qty, optional price override, blue Add — same outcome as the modal once ID and qty are filled.

Why: Links the sale to real stock. The system copies the part’s asking price and VAT % at add time (unless you override price).

When: Draft only. Part must be Available and quantity must not exceed qty on hand.

How: Get ID from Inventory → All parts or from the URL on Edit part (?id=) → type ID → Add. No separate Save draft for lines — each Add saves immediately.

Figure. Part ID box + qty + Add (and Select item… in the same card area).

Save as: docs/manual_screenshots/invoice-05-add-part-card.png

D

Manual line (labour, fee, etc.)

What: Free-text description, qty, price ex VAT, VAT %, Add manual line.

Why: Not everything is a stock part (delivery, fitting, misc.). These lines do not touch parts stock.

When: Draft only.

How: Fill fields → Add manual line.

E

Finalize invoice

What: Large green Finalize invoice button (draft only).

Why: Locks the sale: assigns invoice number, runs SHGA check if needed, reduces stock / marks parts Sold when quantity goes to zero.

When: Only when customer is set, at least one line exists, and you are sure the sale is correct.

How: Click → confirm browser dialog. If SHGA fails, complete customer docs in Customers first, then try again.

F

SHGA reminder (yellow box, draft)

What: Short note: stripped or not-new-condition parts need buyer ID + proof of address on file. Button Open customers.

Why: Matches South African Second-Hand Goods Act expectation for buyers of certain used/stripped goods. The program blocks finalize until compliance files exist on the selected customer.

When: Whenever you sell affected lines (badge SHGA on line).

How: Edit customer → upload scans → save → return to invoice → finalize.

FIN

After finalize — Payments (ZAR)

What: Shows Remaining (still owed), Paid, list of payment lines, form to Add payment (amount, date, method, ref).

Why: This is your accounts receivable for this invoice: total minus payments. Full payment = remaining goes to R 0.

When: Every time the customer pays (cash, EFT, card). You can log several part-payments.

How: Fill amount (not more than remaining) → Add payment. × on a row soft-removes a wrongly entered payment.

PR

Print · Void (final only)

What: Print uses the browser printer (save PDF if you want). Void invoice marks the invoice void — does not return parts to stock automatically.

Why: Paper trail and rare corrections.

When: Print anytime when final; void only if business rules allow and you will fix stock if needed.

VD

Void state

What: Grey banner explaining the invoice is void; no new payments.

Why: Avoid treating cancelled invoices as active debt.

Quick order of work (happy path)

  1. POS → New sale → draft opens.
  2. A — Customer + dates + Save draft.
  3. S / C / P / D — Add all lines (search modal, part ID, full parts list from modal links, or manual line).
  4. Watch zone B — running total incl. VAT in the Lines header updates after each add.
  5. If any line shows SHGA → F — fix customer docs.
  6. E — Finalize invoice.
  7. FIN — Record Payments until remaining is R 0 (or leave balance for true on-account until they pay).
